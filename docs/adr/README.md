# Architectural decision records

Settled **architectural** decisions, one file each — extracted from `domain-model.md`'s former in-line decision log and **back-filled together**, so ADR cross-references point *backward only* and the file numbers are stable identifiers, not a strict timeline (each ADR's `Status:` line carries the actual decision date).

| ADR | Decision |
|---|---|
| [0001](0001-ledger-is-the-durable-record.md) | The Ledger is the channel's durable record (one Ledger of entries; the old separate "Record" noun folded in). |
| [0002](0002-ledger-entries-are-immutable.md) | Entries are immutable; correct by appending — standing is **derived by folding**, never stored (event-sourcing). |
| [0003](0003-ledger-entry-content-model.md) | One entry envelope + a **closed `kind` set**; verification is itself an entry; kernel `Assertion` is minimal; `Park`/`Falsify` collapsed. |
| [0004](0004-any-member-may-author-any-entry.md) | **Any Member (human or agent) may author any entry**; authority lives at the Gate/Verifier layer, not in authorship. |
| [0005](0005-provenance-ref-uri-plus-digest.md) | A `ProvenanceRef` = URI + optional self-describing **content digest** (drift-detectable). |
| [0006](0006-gate-engine-event-sourced.md) | The **Gate engine** is event-sourced into the Ledger (`Proposal`/`Approval`/`Rejection`; derived `GateStatus`; reject is *sticky*). |
| [0007](0007-routing-stays-out-of-the-kernel.md) | **Routing stays out of the kernel**: it executes an `ApprovalRequirement`; the importable **Rubric** layer (future) decides it. |
| [0008](0008-canonical-entry-serialization-is-jcs-json.md) | A `LedgerEntry`'s **canonical byte form** is **JCS / RFC 8785 JSON** (deterministic by spec, readable/diffable); newtypes re-validate on deserialize. |
| [0009](0009-git-refs-substrate-ndjson-per-author.md) | The **git-refs substrate** stores an append-only **NDJSON log per author** under `refs/junto/<channel>/<author>` (local durable record; forge sync deferred). |
| [0010](0010-canonical-order-and-dedup-by-entry-id.md) | Canonical order is **`(timestamp, author email, entry id)`** — a deterministic *total* order — and projection **dedups by `EntryId`** (substrates may hold duplicates). |
| [0011](0011-sync-is-push-fetch-plus-convergent-union-merge.md) | **Sync** = push/fetch of author refs to any git remote; divergence (same author, two machines) reconciles by a **deterministic union-merge** — set union of immutable entries *is* the merge (no CRDT needed). |
| [0012](0012-mcp-over-http-is-the-first-write-surface.md) | The first write surface is **MCP over streamable HTTP** (`junto serve`, localhost:1727). Identity is claimed, not verified — a recorded dogfood-era limit. *(Its UUIDv5 named-channel clause is superseded by 0014.)* |
| [0013](0013-host-serves-the-read-surface-recall-via-hook.md) | The host serves the **read surface**: an HTML channel page (the first pixel of the one surface) + a markdown `/brief`. Agent **recall is a membership concern** (inject at join time, once modelled); a SessionStart hook is the bridge. |
| [0014](0014-channel-identity-is-minted-names-are-substrate-scoped-labels.md) | **ChannelIds are minted (v4), globally unique**; channels are **repo-agnostic** with one **home substrate**; the **name is a substrate-scoped label** bound to the id by a genesis record when the channel is **opened** (an explicit, recorded act — no implicit create-on-first-write). Supersedes 0012's UUIDv5 clause. |
| [0015](0015-one-host-per-machine-serving-many-substrates.md) | **One host per machine/user** (singleton, port 1727) serving **every registered home substrate** — a machine-local substrate registry (not a channel registry); bare names resolve across substrates, ambiguity errors, ids always resolve. The one surface stays one. |
| [0016](0016-channel-lifecycle-acts-are-ledger-entries.md) | **Channel lifecycle acts are ledger entries**: the genesis is a `ChannelOpened` entry (first in the channel's ledger), not a substrate manifest; rename = a later corrective entry; fork/close anticipated as future entry kinds. Extends 0003's closed kind set. |
| [0017](0017-party-is-a-projection-membership-is-founder-granted.md) | **The Party is a projection; membership is founder-granted**: the opener is the founding member; the founder authors `MemberAdded` entries; **only members' entries project** (set-based check) — non-member entries surface as *unrecognized*, never dropped. Minting also mints a **member code** (machine-local secret, never in the ledger) checked by the host's write surfaces — accident-proofing, not security. `MemberLeft`/roles deferred. |
| [0018](0018-human-surface-is-a-desktop-shell-over-the-host.md) | **The human surface is a desktop shell over the host (Tauri)**: v1 wraps the host's server-rendered pages (one implementation of every pixel); the app owns host lifecycle (spawns `junto serve`, leaves it running on exit); the **host connection is a URL** — remote = SSH-tunneled hosts (VS Code Remote pattern), preserving 0012's localhost posture. Desktop crate quarantined from the kernel workspace. |
| [0019](0019-decision-frames-on-subject-entries.md) | **Decision frames ride on subject entries**: an optional payload field (omitted from canonical bytes when absent) where the proposer articulates the verifier's options — each with the act it performs and a drafted, editable rationale. **Unchosen options are recorded** (alternatives-considered become structural — 0003's anticipated richer shape). Kernel stays permissive; surfaces validate kind-coherence; chosen-vs-drafted rationale divergence is a measurable honesty signal. |
| [0020](0020-agent-sessions-and-artifacts-are-ledger-entries.md) | **Agent Sessions and Artifacts are ledger entries**: `SessionStarted` (subject; intent), `SessionUpdated` (act; state fold, last-applicable-wins), `ArtifactAttached` (act; kind + description + provenance). Artifact **content never enters the ledger** (URI + digest only); artifact **kind is a playbook-supplied string**, not a kernel enum. Extends 0003's closed kind set like 0016 did; high-frequency liveness deferred to the Event layer. |
| [0021](0021-member-codes-guard-agent-surfaces-only.md) | **Member codes guard agent surfaces only** (refines 0017): the human surface (web pages + desktop shell) asks for no code — the host *derives* the author from git config and checks **membership** only; a code the server itself stores proves nothing past the OS boundary. Codes stay required on MCP, where identity is *claimed* and the accident (an agent authoring as someone else) is real. The code input, remember-cookie, and retry page are deleted. |
| [0022](0022-channel-close-and-reopen-are-lifecycle-entries.md) | **Channel close/reopen are lifecycle entries** (extends 0016): `ChannelClosed`/`ChannelReopened` fold into a derived `closed` flag, last-applicable-wins, any member (0004). Closed = **out of the working set** (leaves sidebar/focus board; archived on the index; brief warns agents), never out of the record; the **name stays taken**. No tombstone semantics — synced entries still project. |
| [0023](0023-launching-agent-sessions-oneshot-first-pty-next.md) | **Launching Agent Sessions from the surface**: the **Workspace** (machine-local channel→repos mapping, never ledger content) says where sessions run; invocation is a harness **capability** — `oneshot-exec` v1 (`claude -p`/`--resume`, state in the harness's own storage so host restarts are harmless, `--dangerously-skip-permissions`, funded by the subscription Agent SDK credit) with `pty-drive` expected soon after (interactive harness in a portable-pty: subscription billing, mid-turn steering, harness prompts routed to junto — human-driven only). Steering is recorded as a human-authored `SessionUpdated` note, then transported; capture = memo + git-diff artifacts on completion, never scrollback. No trait extracted until OpenCode (harness #2) lands. |
| [0024](0024-acp-is-the-harness-protocol.md) | **ACP is the harness protocol** (`claude -p` fallback): junto drives harnesses over **ACP** (Agent Client Protocol) as primary — *one* client, many harnesses, branching on the **capability flags** `initialize` returns (constraint #4 realized), which **obviates the per-vendor `AgentHarnessAdapter` trait**. The Claude adapter runs Anthropic's SDK so auth is the **subscription, no API token** (spike-verified token-free); ACP + adapter are Apache-2.0 (clean under #1). **Hybrid**: fall back to `claude -p` on test stub / `JUNTO_HARNESS_PROTOCOL=cli` / no Node / ACP-setup failure; retiring `claude -p` deferred. Minimal hand-rolled client (the `agent-client-protocol` crate is the upgrade path). ACP does **not** fix Windows console flashing (orthogonal to the WSL backend). |
| [0025](0025-align-terminology-on-anthropic-managed-agents.md) | **Align terminology on Anthropic's Managed Agents**: a naming-only sweep (no kernel mechanics change). **Persona → Agent** (the config); **Outcome (produced) → Deliverable** so **Outcome** = the *target* (description + rubric); **Agent Session → Session**; routing "Rubric" (0007) → **Routing Policy**; adopt **Rubric** (=verification criteria), **Grader**, and the grading-loop verbs. The **Verifier** noun retires (a Playbook supplies an Outcome + Rubric; a Grader evaluates). `MemberKind::{Human, Agent}` unchanged — "an agent Member runs an Agent". **Gate** survives (Anthropic has no human-approval checkpoint). Supersedes the naming in 0007/0020; done as a big-bang rename PR before the first Playbook. |
| [0026](0026-routing-policy-resolves-to-auto-or-gate-the-autonomy-envelope.md) | **Routing Policy resolves to auto-or-gate (the autonomy envelope)**: a human authorizes *bounded unattended autonomy* by ratifying the Routing Policy for a region; inside it a Grader-`satisfied` Deliverable **auto-resolves its Gate + emits a notification** ("release notes"), outside it the Gate still pauses for a human. The envelope is **not a new noun** — it's the region of the Routing Policy's input space that resolves to 0007's existing `Auto` `ApprovalRequirement`. New: routing is grade-aware (`Auto` reachable only on `satisfied`), `Auto` carries a notification obligation, and the autonomy-granting policy is human-ratified data. Two invariants: **no self-widening** (editing a Routing Policy always routes to a human) and **grade ≠ consent** (the grade is read, never grants autonomy). Makes the Gate *programmable* (strengthens 0025's governance thesis); kernel-general (a future prod-troubleshooting "ship behind a flag if not risky" for free). Builds on 0006/0007 (realizes its future seam), extends 0025; little new kernel work (`Auto` already exists). |
| [0027](0027-channel-lineage-is-diverge-converge-edge-entries.md) | **Channel lineage is diverge/converge edge entries (a cross-channel DAG)**: a lineage edge is **two entries, one in each endpoint's ledger**, referencing the other channel's id — four self-documenting kinds (`DivergedFrom`/`ChildDiverged`/`ConvergedInto`/`ConvergenceReceived`) normalized into one generic `LineageEdge` projection. **Diverge creates the child** (in the parent's substrate, diverger = founder); **converge lands into an existing channel** and closes the source, gate-checked (no open gates dangle). `at` (divergence point) is optional + unvalidated; **projection tolerates dangling edges**. **No Party inheritance** (refines 0017) — only *context* inherits, one hop, via a read-only brief block. Strip draws real attachment points (baseline fallback for roots/dangling). Cross-machine = **sync-mediated**, not remote-MCP. Decides 0016's anticipated fork/converge family; extends 0003's kind set. |
| [0031](0031-junto-reconciles-against-external-forge-state.md) | **junto reconciles against external forge state (it isn't the only actor)** (principle, build deferred): consequential actions happen outside junto — an operator merges a PR in GitHub and junto's record drifts (a converged side-quest showed open while the work shipped). Neither extreme works (accept drift = the verified-record thesis fails; demand all actions flow through junto = user-hostile). Decision: for forge-owned facts (PR merged/closed) the **forge is source of truth** and junto catches up — on sync (+ startup) it polls the PRs it opened (0030's `GateExecuted` URLs) and a merged one auto-converges its side-quest. Poll via `gh` (fits the localhost/no-webhook posture, 0012), not webhooks. This is the **third reconciliation case**; it splits the family by direction — **egress** (push own work out: 0028 built, 0029/0030 deferred) vs **ingress** (pull external truth in: this ADR) — so they share only a thin **sync/startup trigger** + invariants (idempotent, ownership-bounded), not a body. Supersedes the premature "sweep family" framing (junto-dev `4dd4ccb9`). |
| [0032](0032-mid-turn-steer.md) | **Mid-turn steer (interruptable agent turns)** (implemented): a running agent turn couldn't be interrupted/redirected mid-stream (only the timeout ended it), and steering a live turn was racy (it spawned a second concurrent turn). Fix: a **per-session control channel** (human → running turn, the reverse of the live feed) plus a **steerable prompt loop** — the prompt step `select!`s the update stream against control; on a signal it sends a graceful ACP **`session/cancel`** (the adapter tears down its own child tree, sidestepping the Windows kill-tree problem) and either ends the turn (`Interrupt` → new `Interrupted` end-state, lands `Done`) or **re-prompts in the same live process/ACP session** (`Steer` → full context, no reload). One steer box routes on liveness (live → in-place; landed → existing `session/load` resume); both record the steer note (0023). Per-prompt timeout. CLI fallback degrades to interrupt+resume. Parked alternative: S1 cancel+resume in a fresh process (junto-dev `a907e37a`). |
| [0030](0030-surface-approved-but-unexecuted-actionable-gates.md) | **Surface approved-but-unexecuted actionable gates** (designed, not yet built; motivated by the first push-gate dogfood): approval and execution are decoupled (0029), so an actionable gate that is approved but whose executor never runs (stale binary, wrong host, unhandled `kind`, crash) gives **no signal**. Fix: **projection surfaces the gap** — an `Approved` gate with an executable `kind` and **no recorded execution outcome** becomes a distinct "awaiting execution" attention state (derived from the absence of an outcome, so it catches every silent case; display-only, safe on any host), and the **executor records its outcome against the gate** (success *and* failure) so the check is exact. You can't rely on the executor to report it didn't run. Sweep-based auto-retry stays deferred (0029's double-open hazard). |
| [0029](0029-approved-gates-execute-via-an-app-level-reaction.md) | **Approved gates execute via an app-level reaction (the PR-open executor)**: junto's first "approved gate → perform the action" loop, completing the code-PR push-gate. The reaction lives in the **app's approve handlers** (web/MCP), not the kernel — after recording an `Approval`, if the now-`Approved` proposal is a code-PR "Open the PR" gate, the app pushes the `junto/<session>` branch and opens the PR via the `gh`-backed forge (kernel stays generic, 0007; the pre-autonomy-envelope form of 0026). **Fires on local approvals only** (no synced/multi-host double-open; a reconciliation sweep is deferred). Recognition by an **opaque playbook-supplied `kind` tag on Proposal** (`code-pr.open-pr`; kernel stores but never interprets it — the ArtifactAttached.kind pattern, 0020), not by parsing the action string; state recovered from the **workspace + git config** (branch + base), not a proposal↔session key. Idempotent (session-Done check + `gh`'s duplicate guard); the **PR URL is recorded as the deliverable**; capability-gated (only gate when a forge is available); failures surface (no auto-retry v1). |
| [0028](0028-eventually-consistent-lineage-reconciliation.md) | **Eventually-consistent lineage reconciliation** (serves 0027): the **near side writes unconditionally**; the **far side is enqueued** to a machine-local pending-lineage queue (operational state, never ledger content) and retried idempotently (fixed bytes → 0010 dedup) by a reconciliation pass run **on sync + at startup**, writing directly to the substrate (0017/0021 code already verified). **30-day bound** then drop + warn (near side stays "unresolved"). Cross-machine lineage rides 0011 sync, no new network surface. |
